## 1. 画面・モーダル一覧

| 種別 | 名称 | 概要 | 主な関連機能 |
| --- | --- | --- | --- |
| 画面 | ボード一覧画面 | アプリ起動時の入口。作成済みボードの一覧と、新規ボード作成導線を提供する。 | F-B-01〜05 |
| 画面 | カンバン画面(ボード詳細) | 1 つのボードを開いた状態。横にリストが並び、各リストにカードが縦に積まれる。 | F-L-01, F-L-03, F-L-04, F-D-01〜03 |
| モーダル | カード詳細モーダル | 1 枚のカードのタイトル・説明・優先度・ラベル・期日・チェックリストを閲覧/編集する。 | F-C-02〜04, F-LB-04, F-DD-01〜03, F-CL-01〜06, F-PR-01 |
| モーダル | ラベル管理モーダル | ボード単位のラベル(名前+色)を作成・編集・削除する。 | F-LB-01〜03 |
| ダイアログ | 入力ダイアログ(共通) | 1 行テキスト入力(ボード作成・改名、チェックリスト作成など)で共通利用する。 | F-B-01, F-B-04, F-CL-01〜02 |
| ダイアログ | 削除確認ダイアログ(共通) | ボード/リスト/カード/ラベル/チェックリスト削除時の最終確認に共通利用する。 | F-B-05, F-L-03, F-C-04, F-LB-03, F-CL-03 |

## 2. ボード一覧画面

#### レイアウト案

```
+--------------------------------------------------------+
| TaskManagementForAI                                    |
+--------------------------------------------------------+
| あなたのボード                                          |
|                                                        |
| +------------+  +------------+  +------------+         |
| | 個人タスク  |  | 仕事        |  | 勉強       |         |
| |       [...]|  |       [...]|  |       [...]|         |
| +------------+  +------------+  +------------+         |
|                                                        |
| +------------+                                         |
| | + 新規ボード|                                         |
| +------------+                                         |
+--------------------------------------------------------+
```

#### インタラクション仕様
- ボードカードをクリックするとカンバン画面(§3)へ遷移する。
- 「+ 新規ボード」をクリックすると入力ダイアログ(§6)が開く(F-B-01)。
- 各ボードカードの「…」メニューから「改名」「削除」が可能(F-B-04, F-B-05)。
  - 改名: 入力ダイアログ(§6)で現タイトルを初期値に編集。
  - 削除: 削除確認ダイアログ(§6)。

#### 空状態
ボードが 1 件も存在しない場合、画面中央に「ボードがありません」と新規ボード作成ボタンのみを表示する。

## 3. カンバン画面

#### レイアウト案

```
+----------------------------------------------------------------------+
| [< 一覧へ]  ボード名: 「個人タスク」(クリックで編集)  [ラベル管理]  |
+----------------------------------------------------------------------+
| +---------------+  +---------------+  +---------------+ +----------+ |
| | 未着手    […] |  | 進行中    […] |  | 完了      […] | | + リスト | |
| | [優先度順][期限順]| [優先度順][期限順]| [優先度順][期限順]|         |
| +---------------+  +---------------+  +---------------+ +----------+ |
| |[●青 調査][●赤 重要]| [カードC]    |  | [カードE]     |              |
| |[!高] カードA   |  | [カードD]     |  |               |              |
| | 📅 5/10 ☑2/4  |  | + カードを追加|  | + カードを追加|              |
| | [カードB]     |  |               |  |               |              |
| | + カードを追加|  |               |  |               |              |
| +---------------+  +---------------+  +---------------+              |
+----------------------------------------------------------------------+
```

#### インタラクション仕様
- カードクリックで **カード詳細モーダル(§4)** を開く。
- リストヘッダの「…」メニュー: 「リストを削除」(F-L-03)。
- ボードヘッダ: 「< 一覧へ」(ボード一覧へ戻る)、「ラベル管理」(§5 を開く)。ボードタイトルはクリックでインライン編集できる(F-B-04)。
- ソートバー: 各リストヘッダ直下に「優先度順」「期限順」の 2 ボタンを配置。クリックでそのリスト内のカードを並び替える(F-SR-01, F-SR-02)。セッション中のみ有効。
- 「+ カードを追加」: ボタン位置にインラインフォームが展開し、タイトル・優先度・期限日を入力して「保存」で確定 / Esc またはキャンセルで閉じる(F-C-01)。
- 「+ リスト」: 同様にインライン入力欄(F-L-01)。
- カードに表示する補助情報: 上から順に **ラベル(色背景＋テキストのピルバッジ)** → **優先度バッジ** → **タイトル** → **期日(📅 m/d)** / **チェックリスト進捗(☑ 完了/全件)**。

#### D&D 視覚フィードバック(F-D-01〜03)
- ドラッグ中のカード/リストは半透明で追従し、ドロップ可能位置に **プレースホルダ(空き枠)** が挿入される。
- ドロップ後の並び順・所属リストは即座に画面へ反映され、永続化される(機能要件 §8)。

#### 空状態
- リストが 0 件の場合、ボード中央に「+ リスト」ボタンのみを表示する。
- カードが 0 件のリストには「+ カード追加」のみを表示する(明示メッセージは出さない)。

## 4. カード詳細モーダル

#### レイアウト案

```
+-----------------------------------------------------------+
|  カードA(クリックでインライン編集)                  [×] |
+-----------------------------------------------------------+
|  メインエリア              | サイドバー                  |
|  ----------------------   | --------------------------- |
|  ラベル:                  | 優先度: [高▼]               |
|  [●青 調査][●赤 重要]     | 期日:                       |
|  [+ ラベル付与]            |  📅 2026-05-10 [解除]        |
|                            |  ※日付クリックでピッカー起動 |
|  説明                      | ラベル: [+ ラベル付与]       |
|  +----------------------+ |                             |
|  | (複数行テキストエリア)| |                             |
|  +----------------------+ | [チェックリスト追加]         |
|                            |                             |
|  チェックリスト「準備」     |                             |
|  2/4 (50%) [...]           | ---                         |
|  ☑ 資料を集める             | [カードを削除]               |
|  ☑ アウトラインを書く       |                             |
|  ☐ ドラフトを書く           |                             |
|  ☐ レビューを依頼           |                             |
|  + 項目を追加               |                             |
+-----------------------------------------------------------+
|                              [キャンセル] [保存]          |
+-----------------------------------------------------------+
```

#### 操作仕様

| エリア | 操作 | 関連機能 |
| --- | --- | --- |
| タイトル | クリックでインライン編集。空欄での保存は不可(確定ボタン非活性または自動破棄)。 | F-C-03 |
| 説明 | 複数行テキストエリア。フッターの「保存」ボタンで一括保存。 | F-C-03 |
| 優先度 | サイドバーのセレクトボックスで高・中・低を選択。フッターの「保存」ボタンで一括保存。 | F-PR-01 |
| ラベル | 「+ ラベル付与」でポップオーバーを開き、ボードのラベル一覧から選択/解除。ポップオーバー末尾の「ラベル管理を開く」リンクから §5 へ遷移。 | F-LB-04, F-LB-05 |
| 期日 | 設定済みの場合、日付テキストをクリックで date picker(年月日)を起動。横の「解除」ボタンでクリア。フッターの「保存」ボタンで一括保存。 | F-DD-01, F-DD-02 |
| チェックリスト | 「+ チェックリスト追加」で入力ダイアログ(§6)で作成。各項目はインラインで追加・編集・削除、チェックボックスで完了切替。タイトル右の「…」から改名/削除。上部に「完了/全件 (進捗率%)」を常時表示。 | F-CL-01〜06 |
| 保存 | フッターの「保存」ボタンで、タイトル以外のすべてのフィールド(説明・優先度・期日)を一括保存する。 | F-C-03 |
| 削除 | サイドバー最下部の「カードを削除」で削除確認ダイアログ(§6)。 | F-C-04 |
| 閉じる | フッターの「キャンセル」または「×」ボタンで閉じる。未保存の変更は破棄される。 | — |

## 5. ラベル管理モーダル

#### レイアウト案

```
+--------------------------------------+
|  ラベル管理                     [×] |
+--------------------------------------+
|  ●赤   重要        [編集] [削除]    |
|  ●青   調査        [編集] [削除]    |
|  ●緑   完了        [編集] [削除]    |
|                                      |
|  + ラベル追加                        |
|    名前: [____________________]      |
|    色:  [●][●][●][●][●][●][●][●]   |
|    [キャンセル] [追加]               |
+--------------------------------------+
```

#### 操作仕様
- 開き方: カンバン画面ヘッダの「ラベル管理」ボタン、またはカード詳細モーダル(§4)のリンクから。
- 各行: 色プレビュー / 名前 / [編集] / [削除]。編集はその場でインライン編集(F-LB-02)。
- 「+ ラベル追加」: 名前(必須・空欄不可)+ 色(あらかじめ用意された 6〜10 色のパレットから選択)(F-LB-01)。
- 削除: 削除確認ダイアログ(§6)。確定後、付与済みカードからも自動的に外れる(F-LB-03)。

## 6. 共通ダイアログ仕様

#### 入力ダイアログ
1 行テキスト入力を要する操作で共通利用する。

```
+----------------------------------+
|  [タイトル文言]             [×] |
+----------------------------------+
|  [______________________]        |
|                                  |
|             [キャンセル] [決定] |
+----------------------------------+
```

- 可変項目: タイトル文言、入力欄の初期値、プレースホルダ、決定ボタン文言。
- 空文字では「決定」ボタンを非活性化する。
- 想定利用箇所: ボード作成/改名、チェックリスト作成/改名、ラベル名編集など。

#### 削除確認ダイアログ

```
+----------------------------------------+
|  [対象]を削除しますか?            [×]|
+----------------------------------------+
|  この操作は取り消せません。            |
|  配下の[子要素]もすべて削除されます。  |
|                                        |
|              [キャンセル] [削除]      |
+----------------------------------------+
```

- 「削除」ボタンは強調表示(警告色)。
- 配下要素を持つ場合(ボード/リスト/チェックリスト/付与済みラベル等)は、影響範囲を本文に明記する。

## 8. 期限超過カードの視覚仕様(F-DD-04)

期日テキスト部分の表示色で、期限の状態を 3 段階に区別する。

| 状態 | 色の指針 | 用途 |
| --- | --- | --- |
| 期限超過(本日より前) | 赤系(警告色) | 期限を過ぎたことを強く伝える |
| 本日締切 | 橙系(注意色) | 当日中の対応を促す |
| 期限内(本日より後) / 期日未設定 | 通常テキスト色 | 平常表示 |

具体的なカラーコードは設計フェーズで確定するが、「赤=超過」「橙=本日」「通常=未到来」の 3 段階区分は本書で確定する。
