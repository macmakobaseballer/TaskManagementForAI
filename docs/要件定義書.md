## 1. はじめに

### 1.1 背景・目的

個人で抱える日々のタスク・作業項目を、Trello のような **カンバン形式** で視覚的に管理したい。付箋を貼り替える感覚で、タスクの状態(例: 未着手 / 進行中 / 完了)を直感的に動かせる、シンプルな個人用ツールを作る。

### 1.2 用語定義

| 用語 | 定義 |
| --- | --- |
| **ボード (Board)** | 1 つのテーマ・プロジェクトに対応する作業空間。複数のリストを内包する。 |
| **リスト (List)** | ボード内の縦の列。タスクの状態や分類(例:「未着手」「進行中」「完了」)を表す。複数のカードを内包する。 |
| **カード (Card)** | 1 つのタスク。リスト上に縦に積まれる。タイトル(必須)・説明(任意)に加え、ラベル / 期日 / チェックリストをメタデータとして持てる。 |
| **ラベル (Label)** | カードに付与する分類用のタグ。色と名前を持ち、ボード単位で定義する。 |
| **期日 (Due Date)** | カードの締切日。1 件のカードに 1 つ設定できる。 |
| **チェックリスト (Checklist)** | カード内に作るサブタスク群。各項目は完了/未完了の状態を持つ。 |
| **MVP** | 最小限の価値を提供できる最初のバージョン。本書はこの範囲のみを対象とする。 |

### 1.3 本書のスコープ

本書は **個人利用** を前提とした **MVP の範囲** のみを対象とする。
マルチユーザー機能、コメント、添付ファイル、AI 連携などは [7. スコープ外](#7-スコープ外) に列挙のとおり、本書では扱わない。

## 2. 利用者・利用シナリオ

### 2.1 想定利用者

- 単一の個人ユーザー(本人のみ)。
- IT リテラシ: 一般的な Web サービス(Trello / Notion など)を利用した経験のあるレベル。

### 2.2 利用シーン

ユーザーは PC のブラウザでアプリを開き、自分のタスクをボード/リスト/カードの 3 階層で整理する。マウス操作でカードをドラッグして列を移すことで、進捗状態を更新する。

### 2.3 主要ユースケース

- UC-01: ボードを作成して、テーマごとに作業空間を分ける。
- UC-02: ボード内にリスト(例:「未着手」「進行中」「完了」)を作成する。
- UC-03: タスクをカードとしてリストに追加する。
- UC-04: カードをドラッグして別のリストに移動し、進捗を更新する。
- UC-05: 同一リスト内でカードを並べ替え、優先度を表現する。
- UC-06: 不要になったボード / リスト / カードを削除する。
- UC-07: ブラウザを閉じて再度開いても、前回の状態がそのまま復元される。

## 3. 機能要件

### 3.1 ボード管理

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-B-01 | ボード作成 | ユーザーは新しいボードをタイトルを指定して作成できる。 |
| F-B-02 | ボード一覧 | 作成済みボードを一覧表示できる(タイトルが見える)。 |
| F-B-03 | ボード閲覧 | 一覧から 1 つを選択し、そのボードのカンバンを開ける。 |
| F-B-04 | ボード改名 | ボードのタイトルを変更できる。 |
| F-B-05 | ボード削除 | ボードを削除できる(配下のリスト・カードも併せて削除される。誤操作防止のため確認を求める)。 |

### 3.2 リスト管理

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-L-01 | リスト作成 | 開いているボード内に、新しいリストをタイトル付きで作成できる。 |
| F-L-02 | リスト改名 | リストのタイトルを変更できる。 |
| F-L-03 | リスト削除 | リストを削除できる(配下のカードも併せて削除される。確認を求める)。 |
| F-L-04 | リスト順序変更 | ボード内のリストの左右の並び順を変更できる(D&D による。3.4 参照)。 |

### 3.3 カード管理

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-C-01 | カード作成 | リスト内に新しいカードをタイトル付きで作成できる。 |
| F-C-02 | カード閲覧 | カードの詳細(タイトル・説明)を表示できる。 |
| F-C-03 | カード編集 | カードのタイトル(必須)・説明(任意)を編集できる。 |
| F-C-04 | カード削除 | カードを削除できる(確認を求める)。 |

カードのデータ項目は以下のとおり。

| 項目 | 必須 | 内容 |
| --- | --- | --- |
| タイトル | 必須 | 1 行のテキスト。空文字は不可。 |
| 説明 | 任意 | 複数行のテキスト。未入力可。 |
| ラベル | 任意 | 0 件以上のラベル(ボード内で定義されたものから選択)。詳細は §3.5。 |
| 期日 | 任意 | 1 つの年月日。未設定可。詳細は §3.6。 |
| チェックリスト | 任意 | 0 件以上のチェックリスト。各リストは 0 件以上の項目を持つ。詳細は §3.7。 |

### 3.4 ドラッグ&ドロップ

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-D-01 | カードの同一リスト内並べ替え | 1 枚のカードをドラッグし、同じリスト内の任意の位置に挿入できる。 |
| F-D-02 | カードのリスト間移動 | 1 枚のカードをドラッグし、別のリストの任意の位置に挿入できる。 |
| F-D-03 | リストの並べ替え | リストのヘッダ部分をドラッグし、ボード内のリストの左右順を入れ替えられる。 |

ドロップ後の並び順・所属リストは即座に画面へ反映され、永続化される(§3.8 参照)。

### 3.5 ラベル管理

ラベルは **ボード単位** で定義され、そのボード内のカードに付与できる。

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-LB-01 | ラベル作成 | ボードごとにラベル(名前 + 色)を作成できる。 |
| F-LB-02 | ラベル編集 | ラベルの名前・色を変更できる(変更は付与済みカードにも即時反映)。 |
| F-LB-03 | ラベル削除 | ラベルを削除できる。削除されたラベルは付与済みのカードからも自動的に外れる(確認を求める)。 |
| F-LB-04 | カードへの付与・解除 | カードに 0 件以上のラベルを付与/解除できる。1 枚のカードに複数のラベルを付けられる。 |
| F-LB-05 | カード上での視認 | ボード上のカードに、付与されているラベル(色付きのバー/タグ)が視認できる形で表示される。 |

### 3.6 期日

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-DD-01 | 期日設定 | カードに 1 つの期日(年月日)を設定できる。 |
| F-DD-02 | 期日変更・解除 | 設定済みの期日を変更、または解除(未設定に戻す)できる。 |
| F-DD-03 | 期日表示 | カード上に期日が表示される。 |
| F-DD-04 | 期限超過の強調 | 当日を過ぎた期日(期限超過)のカードは、視覚的に強調表示される(色やアイコンで区別)。 |

### 3.7 チェックリスト

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-CL-01 | チェックリスト作成 | カード内にチェックリスト(タイトル付き)を 0 件以上作成できる。 |
| F-CL-02 | チェックリストのタイトル編集 | チェックリストのタイトルを変更できる。 |
| F-CL-03 | チェックリスト削除 | チェックリスト全体を削除できる(配下の項目も併せて削除される)。 |
| F-CL-04 | 項目の追加・編集・削除 | 各チェックリストに項目(テキスト)を追加・編集・削除できる。 |
| F-CL-05 | 項目の完了切替 | 項目をチェック(完了)/アンチェック(未完了)できる。 |
| F-CL-06 | 進捗表示 | チェックリストごとに「完了項目数 / 全項目数」または進捗率(%)が表示される。 |

### 3.8 永続化

| ID | 機能 | 内容 |
| --- | --- | --- |
| F-P-01 | 状態の保存 | ボード / リスト / カード、およびカードのメタデータ(タイトル / 説明 / ラベル / 期日 / チェックリスト)と並び順は、ユーザー操作の結果がそのまま保存される。 |
| F-P-02 | 状態の復元 | ブラウザを再読み込み、または閉じて再度開いた場合でも、前回終了時の状態がそのまま復元される。 |

> 保存の具体的な実装手段(ブラウザのローカルストレージ、IndexedDB、ローカルの簡易 DB など)は **設計フェーズで決定** する。本書では「失われずに保存・復元されること」のみを要件とする。

## 4. 画面要件

### 4.1 画面・モーダル一覧

| 種別 | 名称 | 概要 | 主な関連機能 |
| --- | --- | --- | --- |
| 画面 | ボード一覧画面 | アプリ起動時の入口。作成済みボードの一覧と、新規ボード作成導線を提供する。 | F-B-01〜05 |
| 画面 | カンバン画面(ボード詳細) | 1 つのボードを開いた状態。横にリストが並び、各リストにカードが縦に積まれる。 | F-L-01〜04, F-D-01〜03 |
| モーダル | カード詳細モーダル | 1 枚のカードのタイトル・説明・ラベル・期日・チェックリストを閲覧/編集する。 | F-C-02〜04, F-LB-04, F-DD-01〜03, F-CL-01〜06 |
| モーダル | ラベル管理モーダル | ボード単位のラベル(名前+色)を作成・編集・削除する。 | F-LB-01〜03 |
| ダイアログ | 入力ダイアログ(共通) | 1 行テキスト入力(ボード作成・改名、リスト改名、チェックリスト作成など)で共通利用する。 | F-B-01, F-B-04, F-L-02, F-CL-01〜02 |
| ダイアログ | 削除確認ダイアログ(共通) | ボード/リスト/カード/ラベル/チェックリスト削除時の最終確認に共通利用する。 | F-B-05, F-L-03, F-C-04, F-LB-03, F-CL-03 |

### 4.2 ボード一覧画面

#### レイアウト案

```
+--------------------------------------------------------+
| TaskManagementForAI                                    |
+--------------------------------------------------------+
| あなたのボード                                          |
|                                                        |
| +------------+  +------------+  +------------+         |
| | 個人タスク  |  | 仕事        |  | 勉強       |         |
| |       [...]|  |       [...]|  |       [...]|         |
| +------------+  +------------+  +------------+         |
|                                                        |
| +------------+                                         |
| | + 新規ボード|                                         |
| +------------+                                         |
+--------------------------------------------------------+
```

#### インタラクション仕様
- ボードカードをクリックするとカンバン画面(§4.3)へ遷移する。
- 「+ 新規ボード」をクリックすると入力ダイアログ(§4.6)が開く(F-B-01)。
- 各ボードカードの「…」メニューから「改名」「削除」が可能(F-B-04, F-B-05)。
  - 改名: 入力ダイアログ(§4.6)で現タイトルを初期値に編集。
  - 削除: 削除確認ダイアログ(§4.6)。

#### 空状態
ボードが 1 件も存在しない場合、画面中央に「ボードがありません」と新規ボード作成ボタンのみを表示する。

### 4.3 カンバン画面

#### レイアウト案

```
+----------------------------------------------------------------------+
| [< 一覧へ]  ボード名: 「個人タスク」  [改名] [削除] [ラベル管理]     |
+----------------------------------------------------------------------+
| +---------------+  +---------------+  +---------------+ +----------+ |
| | 未着手    […] |  | 進行中    […] |  | 完了      […] | | + リスト | |
| +---------------+  +---------------+  +---------------+ +----------+ |
| | ●青 ●赤       |  | [カードC]     |  | [カードE]     |              |
| | カードA        |  | [カードD]     |  |               |              |
| | 📅 5/10 ☑2/4  |  | + カード追加  |  | + カード追加  |              |
| | [カードB]     |  |               |  |               |              |
| | + カード追加  |  |               |  |               |              |
| +---------------+  +---------------+  +---------------+              |
+----------------------------------------------------------------------+
```

#### インタラクション仕様
- カードクリックで **カード詳細モーダル(§4.4)** を開く。
- リストヘッダの「…」メニュー: 「リストを改名」「リストを削除」(F-L-02, F-L-03)。
- ボードヘッダ: 「< 一覧へ」(ボード一覧へ戻る)、「改名」「削除」、「ラベル管理」(§4.5 を開く)。
- 「+ カード追加」: ボタン位置にインライン入力欄が開き、タイトルを入力し Enter で確定 / Esc でキャンセル(F-C-01)。
- 「+ リスト」: 同様にインライン入力欄(F-L-01)。
- カードに表示する補助情報: 上から順に **ラベル(色チップ)** → **タイトル** → **期日(📅 m/d)** / **チェックリスト進捗(☑ 完了/全件)**。

#### D&D 視覚フィードバック(F-D-01〜03)
- ドラッグ中のカード/リストは半透明で追従し、ドロップ可能位置に **プレースホルダ(空き枠)** が挿入される。
- ドロップ後の並び順・所属リストは即座に画面へ反映され、永続化される(§3.8)。

#### 空状態
- リストが 0 件の場合、ボード中央に「+ リスト」ボタンのみを表示する。
- カードが 0 件のリストには「+ カード追加」のみを表示する(明示メッセージは出さない)。

| 削除 | 「カードを削除」で削除確認ダイアログ(§4.6)。 | F-C-04 |
| 閉じる | 「×」または背景クリックで閉じる(編集中の内容は自動保存済み前提)。 | — |

### 4.5 ラベル管理モーダル

#### レイアウト案

```
+--------------------------------------+
|  ラベル管理                     [×] |
+--------------------------------------+
|  ●赤   重要        [編集] [削除]    |
|  ●青   調査        [編集] [削除]    |
|  ●緑   完了        [編集] [削除]    |
|                                      |
|  + ラベル追加                        |
|    名前: [____________________]      |
|    色:  [●][●][●][●][●][●][●][●]   |
|    [キャンセル] [追加]               |
+--------------------------------------+
```

#### 操作仕様
- 開き方: カンバン画面ヘッダの「ラベル管理」ボタン、またはカード詳細モーダル(§4.4)のリンクから。
- 各行: 色プレビュー / 名前 / [編集] / [削除]。編集はその場でインライン編集(F-LB-02)。
- 「+ ラベル追加」: 名前(必須・空欄不可)+ 色(あらかじめ用意された 6〜10 色のパレットから選択)(F-LB-01)。
- 削除: 削除確認ダイアログ(§4.6)。確定後、付与済みカードからも自動的に外れる(F-LB-03)。

### 4.6 共通ダイアログ仕様

#### 入力ダイアログ
1 行テキスト入力を要する操作で共通利用する。

```
+----------------------------------+
|  [タイトル文言]             [×] |
+----------------------------------+
|  [______________________]        |
|                                  |
|             [キャンセル] [決定] |
+----------------------------------+
```

- 可変項目: タイトル文言、入力欄の初期値、プレースホルダ、決定ボタン文言。
- 空文字では「決定」ボタンを非活性化する。
- 想定利用箇所: ボード作成/改名、リスト改名、チェックリスト作成/改名、ラベル名編集など。

#### 削除確認ダイアログ

```
+----------------------------------------+
|  [対象]を削除しますか?            [×]|
+----------------------------------------+
|  この操作は取り消せません。            |
|  配下の[子要素]もすべて削除されます。  |
|                                        |
|              [キャンセル] [削除]      |
+----------------------------------------+
```

- 「削除」ボタンは強調表示(警告色)。
- 配下要素を持つ場合(ボード/リスト/チェックリスト/付与済みラベル等)は、影響範囲を本文に明記する。

### 4.7 状態別表示

| 状態 | 表示仕様 |
| --- | --- |
| 空状態(ボード) | §4.2 参照。 |
| 空状態(リスト/カード) | §4.3 参照。 |
| 入力バリデーションエラー | カードタイトル等の必須欄が空欄の場合、確定ボタンを非活性化する。エラーメッセージ表示は任意。 |
| D&D 中 | §4.3 参照。 |
| 永続化失敗 | MVP ではローカル保存前提のため通常運用での失敗は想定しない。失敗が起こりうる構成を採用する場合は、設計フェーズで通知 UI を別途定義する。 |

### 4.8 期限超過カードの視覚仕様(F-DD-04)

期日テキスト部分の表示色で、期限の状態を 3 段階に区別する。

| 状態 | 色の指針 | 用途 |
| --- | --- | --- |
| 期限超過(本日より前) | 赤系(警告色) | 期限を過ぎたことを強く伝える |
| 本日締切 | 橙系(注意色) | 当日中の対応を促す |
| 期限内(本日より後) / 期日未設定 | 通常テキスト色 | 平常表示 |

具体的なカラーコードは設計フェーズで確定するが、「赤=超過」「橙=本日」「通常=未到来」の 3 段階区分は本書で確定する。

## 5. 非機能要件

### 5.1 動作環境

- 主要モダンブラウザの最新安定版(Google Chrome / Microsoft Edge)で動作する。
- デスクトップ画面(横幅 1280px 以上)を主たる対象とする。スマートフォン最適化は本 MVP では対象外。

### 5.2 性能

- 個人利用を想定し、カードの総数が数百程度までは、操作(作成・編集・D&D・画面遷移)に体感的なストレスが出ないこと。
- D&D 時のカード移動は、ドロップから画面反映までが瞬時に感じられること(目安: 100ms 以内)。

### 5.3 セキュリティ

- 個人利用のため、認証(ログイン)機能は MVP では設けない。
- 機微情報を扱う想定はしない。ユーザーは自身のデータを自身の端末で管理する前提。

### 5.4 可用性・保守性

- 外部サービスへの依存は最小化することが望ましい(設計時の指針)。
- 単純な構成を優先し、後続フェーズで機能を増やせる余地を残す。

---

## 6. 制約・前提条件

- **単一ユーザー前提**: 同時に複数人が編集する状況は考慮しない。
- **技術スタック**: 本要件定義段階では未確定。後続の設計フェーズで選定する。
- **デプロイ形態**: 本書では問わない(ローカル起動でも、将来的なホスティングでも要件を満たせる範囲とする)。

---

## 7. スコープ外

以下は **本書では扱わない**(「将来検討」ではなく、本 MVP 要件には含めない)。

- マルチユーザー / ボード共有 / 権限管理 / 招待
- コメント / 添付ファイル(カードに紐付ける汎用ファイル添付)/ メンバーアサイン / 通知
- カード**カバー画像**(色・画像いずれも)
- 検索 / フィルタ / 並び替え条件の保存
- AI 連携(タスクの自動分類、要約、自然言語によるタスク追加など)
- モバイル / デスクトップアプリ(Electron など)としての提供
- 端末間同期 / クラウドバックアップ

## 8. 受け入れ基準

以下のシナリオが、すべて成功裏に実行できることを MVP のリリース基準とする。

1. ユーザーは新規ボード「個人タスク」を作成できる。
2. ユーザーは作成したボードを開き、リスト「未着手」「進行中」「完了」を順に作成できる。
3. ユーザーは「未着手」リストにカード「資料を作成する」を追加できる。
4. ユーザーは作成したカードのタイトルと説明を編集できる。
5. ユーザーはそのカードを「進行中」リストに **ドラッグ&ドロップ** で移動できる。移動後、画面に即座に反映される。
6. ユーザーは「進行中」リスト内で複数カードの並び順を入れ替えられる。
7. ユーザーはボード内のリストの左右順を入れ替えられる。
8. ユーザーは色付きラベル(例: 赤「重要」/ 青「調査」)をボードに作成し、複数のカードに付与/解除できる。付与状況はカード上の見た目で確認できる。
9. ユーザーはカードに期日(例: 翌日)を設定でき、期日を過ぎたカードは視覚的に強調表示される。
10. ユーザーはカード詳細でチェックリストを作成し、項目を追加・チェック・アンチェックできる。完了/全件の進捗が表示される。
11. ユーザーがブラウザを **再読み込み** し、また一度閉じて開き直しても、上記すべての状態(ボード・リスト・カードの存在および並び順、タイトル・説明、ラベル付与、期日、チェックリストとその完了状態)が保持されている。
12. 上記 1〜11 が、Chrome / Edge の最新安定版でいずれも動作する。
